# 📊 Desafio1 Ciencia de Datos Alura Store.

## 📘 Descripción del Proyecto
Este proyecto desarrolla un análisis integral del desempeño financiero, operativo y logístico de cuatro tiendas de una cadena comercial.  
El análisis se implementa en **Python**, integrando análisis descriptivo, visualizaciones y modelado financiero para fundamentar una decisión estratégica.

---

## 🎯 Objetivos

### Objetivo General
Realizar un analisis de ventas para evaluar la rentabilidad, eficiencia logística y desempeño de cada tienda para recomendar al señor Juan a que tome una decisión estratégica informada sobre cuál debería ser vendida para que pueda invertir en un negocio nuevo.

### Objetivos Específicos

- 💰 **Financiero:** Analizar los ingresos y márgenes por tienda.
- 🚚 **Logístico:** Calcular y comparar los costos de envío y el tiempo de despacho.
- ⭐ **Cliente:** Evaluar la satisfacción mediante calificaciones promedio.
- 🗺️ **Geográfico:** Examinar la distribución de ventas y su impacto en la operación.
- 📉 **Decisión:** Construir indicadores consolidados para apoyar la recomendación final.
  
---

## 🧠 Alcance
El análisis abarca datos de ventas, precios, calificaciones, costos logísticos y coordenadas geográficas.
* **Fuente de Datos:** Los datasets se consumen directamente desde un repositorio externo (Git).
* **Procesamiento:** Limpieza, validación y transformación de datos en el entorno de **Google Colab**.
* **Entregable:** Un reporte técnico con la recomendación final.

## 🚀 Instrucciones de Uso (Reproducibilidad)
Para replicar este análisis de manera auditable:
1. Abre el archivo notebooks/AluraStoreLatam.ipynb en Google Colab.
2. Ejecuta la primera celda para instalar las dependencias listadas en requirements.txt.
3. Ejecuta la segunda sección para cargar los datos directamente desde el repositorio fuente externo (URL).
4. Ejecuta todas las celdas secuencialmente ("Run All") para regenerar las métricas, visualizaciones y el reporte final.

## 📊 Resultados Clave y Conclusiones

Basado en un análisis integral, si el objetivo es reducir la complejidad operativa, mejorar la eficiencia logística y elevar la satisfacción promedio del cliente en el portafolio restante, se recomienda considerar la venta de la Tienda 1. Aunque representa la mayor facturación, sus ineficiencias inherentes podrían ser una carga a largo plazo.

Si la prioridad es mantener el mayor volumen de ingresos posible y se confía en poder mejorar la operación de la Tienda 1, entonces la Tienda 4 sería la candidata a vender, ya que es la de menor escala.

Sin embargo, los datos sugieren que la Tienda 1 presenta los desafíos más pronunciados en áreas clave de costo y satisfacción, lo que la convierte en una candidata lógica para desinversión si se busca optimizar el rendimiento general de la red de tiendas.
